---
title: "Glosario"
description: "Diccionario rápido de los términos que usamos en Puente OS."
---

| Término | Definición |
|---|---|
| **Workspace** | Contenedor principal de tu organización. Una empresa = un workspace. |
| **Team** | Sub-grupo dentro del workspace, típicamente un departamento o squad. |
| **Member** | Persona con acceso al workspace. Puede pertenecer a uno o varios teams. |
| **Project** | Iniciativa operativa concreta. Contenedor donde conviven tasks y construcción. |
| **Initiative** | Objetivo estratégico que cruza varios teams y projects. |
| **Task** | Unidad atómica de trabajo operativo. Tiene estado, prioridad, asignado y fecha límite. Es el término canónico de Puente OS — el lente del operador, no del bug tracker. |
| **Sub-task** | Task hija de otra Task. Para descomponer trabajo. |
| **Cycle** | Período recurrente (1-8 semanas) donde el team enfoca su trabajo. |
| **View** | Lente dinámica para filtrar y ordenar Tasks o Builds según criterios. |
| **Strategic Map** | Norte + métricas que define el cliente al inicio del Onboarding. |
| **Strategic Discovery** | Entry point pago (~1 semana) que produce el Strategic Map. |
| **Onboarding** | Las 4 fases bundled (Strategic Discovery → Initiatives Definition → Projects Mapping → Builds Scoping). 4-6 semanas. |
| **Build Cycle** | Unidad rítmica continua post-Onboarding (mensual default). Cada Build Cycle entrega N Builds promovidos a Live. |
| **Build** | Artefacto de software construido en un Project. Hay 4 tipos: App, Agent, Workflow, Database. |
| **App** | Interfaz web construida con App Builder. La capa visual con la que interactúa el usuario. |
| **Database** | Almacenamiento estructurado del cliente con schema, permisos y vistas. El término canónico — el módulo en la UI se llama Tables. |
| **Workflow** | Automatización declarativa con triggers, condiciones, código Python y agentes. |
| **Agent** | Entidad agéntica con propósito específico. Puede ser embebida como widget conversacional. |
| **Trigger** | Evento que dispara un workflow (ej. cambio en una fila, llegada de un email). |
| **Node** | Cada paso dentro de un workflow. Puede ser de tipo acción, código, agente o trigger. |
| **Scenario** | Otra forma de llamar a un workflow. Tienen el mismo significado en la interfaz. |
| **Schema** | Estructura de columnas (nombre + tipo) de una Database. |
| **Prompt** | Texto que le das al Agent o al App Builder para que genere algo. |
| **Agent Guidance** | Convenciones, integraciones autorizadas y reglas con las que opera un Agent. |
| **Modo Inteligente** | Modo de configuración del Agent donde IA genera el prompt por ti. |
| **Modo Manual** | Modo de configuración donde escribes el prompt directamente. |
| **Modo Inspect** | En App Builder, modo donde puedes editar la app haciendo click en cualquier elemento. |
| **Preview** | Vista de la app generada en tiempo real, antes de publicar. |
| **Theme** | Paleta de colores, tipografías y bordes que aplica a las apps. |
| **Widget** | Componente embebible (típicamente el chatbot del Agent) en cualquier sitio externo. |
| **Variable `{{ }}`** | Sintaxis para pasar datos entre nodos en un workflow. |
| **Bookmark** | Nodo o configuración guardada para reutilizar en otros workflows. |
| **Triage** | Inbox donde llegan tasks sin asignar para revisar y rutear. |
