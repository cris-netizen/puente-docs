Puente OS NO reemplaza las herramientas que tu equipo ya usa — **elimina el trabajo manual entre ellas**. Cualquier sistema con API REST se conecta via nodo HTTP de [Workflows](/componentes/workflows). Para casos comunes ofrecemos nodos nativos con autenticacion y formularios pre-configurados.

<Note>
  **Como leer los badges:**
  - `nodo nativo` — integracion lista para usar, sin escribir codigo
  - `via HTTP` — conexion via REST con tu API key, configurable con nodo HTTP en Workflows
  - `via Python` — para logica custom que requiere libreria especifica, usando el nodo Codigo Python
</Note>

## No encuentras tu integracion?

Cualquier sistema que tenga **API REST** se conecta con Puente OS — basta configurar el nodo HTTP en Workflows con tu endpoint y autenticacion.

Si tu sistema tiene una API **inusual** (SOAP, file-based, FTP, propietaria), podemos discutir conector custom. Escribenos a [octavio@puente.xyz](mailto:octavio@puente.xyz) con el caso.
