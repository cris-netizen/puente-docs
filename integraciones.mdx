Puente OS NO reemplaza las herramientas que tu equipo ya usa — **elimina el trabajo manual entre ellas**. Cualquier sistema con API REST se conecta vía nodo HTTP de [Workflows](/componentes/workflows). Para casos comunes ofrecemos nodos nativos con autenticación y formularios pre-configurados.

<Note>
  **Cómo leer los badges:**

  - `nodo nativo` — integración lista para usar, sin escribir código
  - `vía HTTP` — conexión vía REST con tu API key, configurable con nodo HTTP en Workflows
  - `vía Python` — para lógica custom que requiere librería específica, usando el nodo Código Python
</Note>

## ¿No encuentras tu integración?

Cualquier sistema que tenga **API REST** se conecta con Puente OS — basta configurar el nodo HTTP en Workflows con tu endpoint y autenticación.

Si tu sistema tiene una API **inusual** (SOAP, file-based, FTP, propietaria), podemos discutir conector custom. Escríbenos a [octavio@puente.xyz](mailto:octavio@puente.xyz) con el caso.
